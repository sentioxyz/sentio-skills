# Sentio Toolkit

## Overview
Work across Sentio processors, Sentio SQL, and alerts with clear workflows, safe defaults, and references to authoritative docs.

## Workflow
### Step 1: Identify the task
Pick the primary track:
- Processor development (code + bindings)
- Sentio SQL execution (Data Studio)
- Alert setup (metrics/log/sql alerts)

### Step 2: Choose the right reference
- Processors: `references/processor-writer.md`, `references/quickstart.md`, `references/processor-engine.md`, `references/handlers-ctx.md`, `references/metrics.md`, `references/entity-store.md`
- SQL: `references/sql.md`
- Alerts: `references/alerts.md`

### Step 3: Execute the track
- Initialize processor projects via Sentio CLI, then implement handlers and build/upload.
- Run or share SQL in Data Studio and export results if needed.
- Create or update alerts, attach notifications, and validate conditions.

## Processor Track
### Writer contract
When the user asks to write a processor, produce:
- A CLI init command first: `npx @sentio/cli@latest create <project-name> -c <chain-type>`.
- `src/processor.ts` as the primary deliverable.
- `schema.graphql` when persistent entities are needed.
- `src/processor.test.ts` with a minimum config smoke test.
- Build and test commands (`sentio test`, `sentio build`, `sentio upload` when requested).

### Handler flow
- Start from the chain-specific scaffold generated by `sentio create`.
- Bind processors and add handlers for events/calls/intervals.
- Use `ctx.meter`, `ctx.eventLogger`, `ctx.store`, and `ctx.exporter` appropriately.
- Persist state in Entity Store rather than global in-memory state.
- Follow chain-specific templates from `references/processor-writer.md`.

### Build and upload
- Build to generate bindings and validate schema.
- Upload to deploy the processor.

## Sentio SQL Track
### Run SQL in Data Studio
- Use the SQL editor to execute queries and inspect results.
- Save or share queries and use endpoints when needed.
- Use `scripts/list_public_tables.py` to discover public tables and columns.
- Use `scripts/execute_sql.py` to run SQL via API and paginate with cursors.

## Alerts Track
### Create alerts
- Pick alert type (metrics, log, or SQL).
- Define conditions and thresholds.
- Attach notification channels and test if available.

## Pitfalls to avoid
- Avoid global in-memory state between handlers; use Entity Store.
- Avoid high-cardinality metric labels and reserved suffixes/labels.
- Avoid expensive SQL queries without limits.
- Avoid alerts with overly broad conditions or missing notifications.

## Recognize requests
- "Create an ERC20 transfer counter processor."
- "Store user balances in Entity Store."
- "Run this SQL in Sentio Data Studio."
- "Create an alert when this metric exceeds a threshold."
