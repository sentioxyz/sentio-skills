# Processor Engine Notes

## Binding
- Use `bind(address)` to attach a processor to a contract address.
- Use `sentio.yaml` to manage contract metadata and ABI wiring when needed.
- Include chain-specific bounds when needed: `startBlock`, `endBlock`, or `startVersion`.

## Stateless handlers
- Avoid global in-memory state across handlers.
- Persist state in Entity Store via `ctx.store`.

## Processor types
- Use built-in processors for common standards when available.
- For custom contracts, use ABI-based processors generated by the SDK.
- Use templates when a factory deploys many instances that need identical handlers.
- Use object processors for Sui object IDs or object types when the task is object-centric.
- Use global processors when the task is chain-wide transaction aggregation.
